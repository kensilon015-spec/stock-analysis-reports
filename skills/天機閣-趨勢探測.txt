# 天機閣 — 趨勢探測技能

## 技能資訊
- 名稱：天機閣-趨勢探測
- 版本：1.0（骨架版，Phase 3 完善）
- 類型：監控 / 爬蟲 / 快報 / 異常警示
- 觸發條件：由總部調度，或直接指令「最新」「日報」「快報」「監控」

## 角色定位
趨勢探測是天機閣的上游情報來源，負責：
1. 從各管道蒐集最新資訊
2. 依三層監控機制篩選重要情報
3. 產出每日快報摘要
4. 觸發異常警示

## 三層監控機制

### 第一層：關鍵字監控
- 比對各部門設定檔中的「監控關鍵字」
- 來源：新聞、RSS、社群、公告
- 速度最快，涵蓋面最廣

### 第二層：主題規則監控
- 比對各部門設定檔中的「主題規則」
- 跨關鍵字的語意組合判斷
- 範例：「營收」+「衰退」+「連續」→ 觸發金融戰情室

### 第三層：語意監控（LLM）
- 使用 LLM 進行語意理解
- 判斷情報的重要性、相關性、影響程度
- 最精準但速度最慢，用於關鍵情報

## 蒐集管道（Phase 3 實作）

| 管道 | 說明 | 狀態 |
|------|------|------|
| 網路搜尋 | 即時搜尋特定主題 | 可用（透過 AI 工具） |
| RSS 訂閱 | 自動抓取訂閱來源 | Phase 3 待建 |
| 網頁爬蟲 | 定時爬取特定網站 | Phase 3 待建 |
| 社群監控 | PTT、社群平台追蹤 | 待處理清單 |
| 公告監控 | 證交所、金管會等 | Phase 3 待建 |

## 每日快報流程（Phase 3 實作）

```
定時觸發（排程）
  │
  ├─ 1. 各管道爬取最新資訊
  ├─ 2. 三層監控篩選
  ├─ 3. 自動標記可信度（來源信任清單比對）
  ├─ 4. 生成情報卡 → 送藏經閣歸檔
  ├─ 5. 彙整每日快報 HTML
  └─ 6. 存放至 天機閣-趨勢探測/每日快報/
```

## 異常警示機制

| 等級 | 觸發條件 | 動作 |
|------|---------|------|
| 一般 | 單一部門出現重要關鍵字 | 標記情報卡，納入日報 |
| 重要 | 多個關鍵字同時觸發 | 產出專題快報 |
| 緊急 | 多部門同時觸發異常 | 自動啟動推衍流程 |

## 目前可用功能
在 Phase 3 爬蟲架構完成之前，趨勢探測可透過以下方式運作：
1. **手動觸發**：閣主說「收集 XXX 最新資訊」→ 總部調度 → 即時網路搜尋
2. **AI 搜尋**：利用 AI 工具的網路搜尋功能即時取得資訊
3. **使用者提供**：閣主貼上網址或文字 → 解析後生成情報卡
