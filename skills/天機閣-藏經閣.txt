# 天機閣 — 藏經閣技能

## 技能資訊
- 名稱：天機閣-藏經閣
- 版本：1.0
- 類型：歸檔整理 / 索引管理 / 情報卡生成 / 查閱檢索
- 觸發條件：由總部調度，或直接指令「歸檔」「查閱」「索引」

## 角色定位
藏經閣是天機閣的唯一知識中心，負責：
1. 生成情報卡（JSON 儲存 + HTML 呈現）
2. 自動分類歸檔到對應部門資料夾
3. 管理索引頁面（部門索引、子分類索引）
4. 查閱檢索已歸檔的情報

## 功能模式

### 模式一：歸檔（Archive）
觸發詞：「歸檔」「存檔」「整理」「建檔」

**標準作業程序：**
1. 讀取 天機閣-設定/情報卡格式.json 確認欄位規範
2. 讀取 天機閣-設定/歸檔規則.json 判斷歸屬部門與子分類
3. 讀取 天機閣-設定/來源信任清單.json 自動標記可信度星等
4. 生成情報卡 JSON 檔案，存放至對應部門資料夾
5. 生成情報卡 HTML 檔案（使用 天機閣-設定/模板/情報卡模板.html 為範本）
6. 更新對應的索引頁面

**情報卡 ID 格式：** `[日期]_[部門代號]_[流水號]`
- 範例：2026-03-19_finance_001

**歸檔路徑規則：**
```
天機閣-藏經閣/【{部門名稱}】/{子分類}/{情報卡檔名}
```

### 模式二：查閱（Retrieve）
觸發詞：「查」「找」「調閱」「看一下」「列出」

**查閱方式：**
1. 依部門瀏覽：指定部門 → 列出該部門所有情報
2. 依標籤搜尋：跨部門搜尋特定標籤的情報
3. 依時間查詢：指定時間區間的情報
4. 關鍵字搜尋：全文搜尋匹配的情報
5. 依狀態篩選：新鮮 / 有效 / 歷史 / 封存

**查閱結果呈現：**
- 列表模式：標題 + 簡述 + 日期 + 可信度星等
- 點擊展開：完整卷宗時間線

### 模式三：更新（Update）
觸發詞：「更新情報」「追加」「後續發展」

**卷宗追加流程：**
1. 找到既有的情報卡（依 ID 或標題搜尋）
2. 在「卷宗」陣列中追加新的時間線條目
3. 更新「最後更新」時間戳
4. 重新生成 HTML 檔案
5. 更新索引頁面

### 模式四：索引管理（Index）
觸發詞：「更新索引」「重建索引」

**索引更新流程：**
1. 掃描指定部門資料夾下的所有情報卡 JSON
2. 彙整為索引資料
3. 更新對應的索引 HTML 頁面

## 情報卡生成規範

### JSON 儲存格式
依據 天機閣-設定/情報卡格式.json 的欄位定義，每張情報卡包含：
- 基本資訊：id, 標題, 部門, 子分類, 標籤
- 評級：可信度(1-5), 價值度(1-5), 時效性
- 狀態管理：狀態, 關聯情報, 推衍建議
- 閣主標記：重要, 追蹤, 不想看, 備註
- 卷宗：時間線條目陣列（初始 + 追加）

### HTML 輸出格式
使用 天機閣-設定/模板/情報卡模板.html 為基底，動態填入資料：
- 標題區：情報標題 + 部門 + 日期 + 可信度星等
- 標籤列：所有標籤以 tag 樣式呈現
- 卷宗時間線：依時間排列，可展開/收合
- 操作按鈕：重要 / 追蹤 / 不想看 / 備註
- 關聯區：相關情報卡的連結

## 閣主標記按鈕行為

| 按鈕 | 動作 | 寫入位置 |
|------|------|---------|
| 重要 | 標記此情報為重要 | 情報卡.閣主標記.重要 = true + 偏好紀錄.手動標記 |
| 追蹤 | 追蹤此情報後續 | 情報卡.閣主標記.追蹤 = true + 偏好紀錄.手動標記 |
| 不想看 | 標記不感興趣 | 情報卡.閣主標記.不想看 = true + 過濾清單.排除紀錄 |
| 備註 | 閣主手寫備註 | 情報卡.閣主標記.備註 |

## 檔案路徑

- 來源信任清單：天機閣-設定/來源信任清單.json
