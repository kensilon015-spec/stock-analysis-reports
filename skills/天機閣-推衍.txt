# 天機閣 — 推衍技能

## 技能資訊
- 名稱：天機閣-推衍
- 版本：1.0
- 類型：分析引擎（影響鏈分析 / 情境模擬 / 個人化推衍 / 投資分析）
- 版本：1.1
- 觸發條件：由總部調度，或直接指令「分析」「推衍」「推算」「預測」

## 角色定位
推衍部門是天機閣的分析引擎，**只做分析，不儲存資料**：
- 輸入：從藏經閣讀取情報
- 處理：依方法論進行分析推衍
- 輸出：產出報告歸檔到藏經閣/天機錄/

## 推衍方法論
定義檔案：天機閣-推衍/推衍方法論.json

### 方法一：影響鏈分析
事件 → 直接影響 → 間接影響 → 機會/風險

**步驟：**
1. 確認觸發事件（讀取情報卡或閣主指定的事件）
2. 列出直接影響對象（產業、企業、政策、市場）
3. 推導間接影響（二階效應、三階效應）
4. 評估對閣主的機會與風險（交叉比對閣主樓資料）
5. 產出建議行動

**輸出格式：**
```
事件：[觸發事件描述]
├─ 直接影響
│   ├─ [影響對象1]：[影響說明]
│   ├─ [影響對象2]：[影響說明]
│   └─ ...
├─ 間接影響（二階）
│   ├─ [影響對象]：[影響說明]
│   └─ ...
├─ 間接影響（三階）
│   └─ ...
├─ 機會
│   └─ [具體機會與建議行動]
└─ 風險
    └─ [具體風險與應對措施]
```

### 方法二：情境模擬
What-if 分析，模擬不同情境下的發展

**步驟：**
1. 定義基準情境（目前已知的事實狀態）
2. 設計三種情境：
   - 樂觀情境：最好的發展方向 + 觸發條件
   - 中性情境：最可能的發展方向
   - 悲觀情境：最壞的發展方向 + 觸發條件
3. 各情境下的影響推衍
4. 評估各情境發生機率（高/中/低）
5. 產出情境比較報告

**輸出格式：**
```
基準情境：[目前狀態描述]

┌─ 樂觀情境（機率：X%）
│  觸發條件：...
│  發展推衍：...
│  對閣主影響：...
│  建議行動：...
│
├─ 中性情境（機率：X%）
│  發展推衍：...
│  對閣主影響：...
│  建議行動：...
│
└─ 悲觀情境（機率：X%）
   觸發條件：...
   發展推衍：...
   對閣主影響：...
   建議行動：...
```

### 方法三：個人化推衍
調取閣主樓資料，分析對閣主的具體影響

**步驟：**
1. 讀取閣主持股（投資組合與持股/）
2. 讀取閣主興趣（閣主興趣/興趣清單.json）
3. 讀取決策紀錄（決策日誌/）
4. 交叉比對事件與閣主資料
5. 推算對閣主的具體影響（持股、興趣、決策）
6. 產出個人化建議

### 方法四：投資分析
台股個股深度分析，8 步驟全流程，整合 Iker 四層評審框架。

**獨立技能檔**：`天機閣/Skills/天機閣-推衍/投資分析.skill`
**評審團技能**：`天機閣/Skills/天機閣-推衍/評審團.skill`

**步驟：**
1. 資料收集（年報 + AI 搜尋 + 市場調查五條線 + finlab 量化 + 閣主偏好）
2. FIAS 資料分級與整合（事實/推論/假設/推測 + 會計品質掃描）
3. 競爭力分析（競爭力地圖 7 維度 + 護城河框架 + 產業面 + 籌碼分析）
4. ACH 競爭假說分析（≥4 假說 + 證據矩陣 + 最少反證原則）
5. 假設壓力測試（關鍵假設檢查 + 事前驗屍 + 敏感度 + 盲點聲明）
6. 評審團（Iker 四層框架 23 項 × 5 角色逐層推衍）
7. 決策輸出（主假說 + 信心等級 + 估值 + 行動建議）
8. 追蹤回饋設定（監控指標 + 命中率統計）

**觸發條件：**
- 閣主直接指令：「研究 XXX」「XXX 可不可以買」「XXX 有沒有風險」
- 推衍內部調用：推衍過程中碰到個股標的
- 戰情室接力：收集完情報後閣主要求深入分析

**資料歸檔**：天機閣-藏經閣/投資分析資料/{代號}_{公司名}/（個股專屬資料夾）
**最終報告**：天機閣-藏經閣/天機錄/{日期}_投資分析_{公司名}{代號}.html

**設定檔**：天機閣-設定/投資分析/（10 個 JSON 設定檔）

**員工知識庫**：天機閣-推衍/員工/Iker/（Iker 個股研究系統）

## 推衍報告產出流程

```
閣主指令 / 情報卡觸發推衍
  │
  ├─ 1. 從藏經閣讀取相關情報
  ├─ 2. 選擇推衍方法（或自動判斷）
  ├─ 3. 執行推衍分析
  ├─ 4. 如需補充資料 → 即時搜尋最新資訊
  ├─ 5. 產出推衍報告（HTML 格式）
  ├─ 6. 報告歸檔到 藏經閣/天機錄/
  └─ 7. 呈現給閣主
```

## 推衍報告命名規範
`[日期]_推衍_[主題].html`
範例：`2026-03-19_推衍_AI晶片禁令.html`

對應 JSON：`[日期]_推衍_[主題].json`

## 注意事項
- 推衍過程中如果缺乏具體數據，必須明確說明，不要編造數字
- 機率評估如果無法量化，用「高/中/低」定性描述
- 所有推衍結論都應註明推論依據，方便閣主判斷
- 推衍部門本身不留存任何資料，所有產出都歸檔到藏經閣
