# 投資分析模組 — 推衍系統/投資分析引擎

## 技能資訊
- 名稱：投資分析
- 版本：1.5
- 類型：深度分析引擎（台股個股研究 8 步驟全流程，整合 OSINT/推衍/股價合理性分析）
- 隸屬：推衍系統子模組
- 觸發條件：閣主直接指令 / 推衍內部調用 / 戰情室接力
- 設定檔路徑：天機閣-設定/投資分析/

## 角色定位
投資分析模組是推衍系統的子模組，專門處理台股投資分析需求。
從事實出發，整合市場調查、情報分析方法論、多假說驗證，
產出可追溯、可驗證、可回測的投資研究報告。

## 核心原則
- **基於事實推論**：每一個輸出都能回答「你怎麼知道的？」
- **禁止觀點句**：禁止「我認為」「我覺得」「應該會」，只允許「事實顯示」「數據指出」「推論鏈為」
- **推論鏈完整性**：結論必須能沿著事實推論鏈追溯到原始資料來源
- **外部驗證優先**：公司自述需要外部驗證，客戶/供應商的數據更可信
- **反證比正證重要**：選「最少反證」的假說，不是「最多正證」的假說
- **假設透明化**：所有假設明確列出、分級、壓力測試
- **FIAS 標記**：每條資訊標記事實等級（F 事實 / I 推論 / A 假設 / S 推測）

## 八步驟全流程

```
閣主指令（股票代號 / 公司名 / 投資問題）
  │
  ├─ 第一步：資料收集（五管道並行）
  │   ├─ A. 年報解析（主動從公開資訊觀測站取得）
  │   ├─ B. AI 必搜清單（使用 WebSearch 搜尋法說會/營收/新聞/產業報告）
  │   ├─ C. 市場調查五條線（客戶/供應商/對手/第三方/人才）
  │   ├─ D. finlab 量化取數（直接執行程式碼抓取，不等待）
  │   └─ E. Claude 即時網路搜尋（WebSearch + WebFetch，貫穿全流程）
  │
  ├─ 第二步：資料分級與整合
  │   ├─ FIAS 四級分類（事實/推論/假設/推測）
  │   ├─ 來源可靠度評分（A~E）
  │   ├─ 會計品質掃描（M-Score / Z-Score / 台股紅旗）
  │   └─ 矛盾偵測 + 缺口報告
  │
  ├─ 第三步：競爭力分析
  │   ├─ 3a. 競爭力地圖（7 維度事實拼圖）
  │   ├─ 3b. 護城河框架（角色定位/定價能力/市佔率/總評級）
  │   ├─ 3c. 產業面分析（生命週期/五力/定位矩陣）
  │   └─ 3d. 籌碼分析（法人/融資券/集中度/內部人/技術面）
  │
  ├─ 第四步：競爭假說分析（ACH 方法）
  │   ├─ 列出 ≥4 假說（含 ≥1 看空）
  │   ├─ 證據矩陣（CC/C/N/I/II）
  │   ├─ 診斷力排序
  │   └─ 最少反證原則 → 選出主要假說
  │
  ├─ 第五步：假設壓力測試
  │   ├─ 關鍵假設檢查（三桶分類：有支撐/需附註/無支撐）
  │   ├─ 事前驗屍法（≥7 個失敗原因 + 風險評分）
  │   ├─ 敏感度分析（找致命因子）
  │   └─ 盲點聲明（誠實列出看不到的東西）
  │
  ├─ 第六步：評審團（調用 評審團.skill）
  │   ├─ Iker 四層框架 23 項逐層推衍
  │   ├─ 五角色觀點整合
  │   └─ 風控官橫切檢查
  │
  ├─ 第七步：決策輸出（質化分析報告，10 大段）
  │   ├─ 一~四：公司概覽 / 為什麼在漲 / 市場忽略的問題 / 隱藏的優勢
  │   ├─ 五~七：六步關鍵因素彙整 / 風險清單與壓力測試 / 監控儀表板
  │   ├─ 八：推衍預測（OSINT 情報 + 短中長期推衍鏈）
  │   ├─ 九：股價合理性分析（五種估值方法交叉比對）
  │   └─ 十：投資結論（三情境估值 + 免責提醒，放最後）
  │
  └─ 第八步：追蹤回饋設定
      ├─ 監控指標 + 失效條件
      ├─ 下次檢查點日期
      └─ 命中率統計回饋後設層
```

## 第一步：資料收集

### 管道 A — 年報 + 公開說明書
年報是最重要的一次來源，**主動取得而非等待閣主提供**。
公開說明書是增資/發行可轉債時的重要資訊來源。

#### A-1. 年報取得方式（依優先序）

**方法一：公司官網投資人專區（優先推薦）**
- 多數上市櫃公司官網「投資人專區 → 財務資訊 → 年報」提供 PDF 下載
- 優點：URL 穩定、可 curl 直接下載、通常有近 5~10 年的歷年年報
- 實測範例（華邦電）：
  ```
  https://www.winbond.com/hq/about-winbond/investor/financial-information/annual-report/
  PDF 路徑：/export/sites/winbond/.../report/{year}-Annual-Report-TC.pdf
  ```
- 注意：各公司 URL 格式不同，需先訪問年報頁面取得實際下載連結
- 注意：不同年度的檔名格式可能不一致（如 2017 年與 2018 年格式不同），需逐一探測

**方法二：公開資訊觀測站（MOPS）**
- https://mops.twse.com.tw → 彙總報告書 → 年報
- 注意：MOPS 有安全性限制，curl 直接存取會被擋（回傳安全錯誤頁面）
- 適合在瀏覽器中手動查詢，程式化存取有困難

**方法三：TWSE 電子資料查詢系統**
- https://doc.twse.com.tw/server-java/t57sb01?step=1&co_id={股票代號}&mtype=A
- 包含 IFRS 財務報告（季報/年報）、完整度最高、可追溯到民國 87 年
- 注意：下載需 JavaScript 互動（session 機制），curl 無法直接取得 PDF
- 適合瀏覽器手動下載，或未來開發 Selenium 自動化工具

**方法四：閣主提供**
- 閣主直接提供年報 PDF 或文字

**年報下載策略**：
1. 先嘗試公司官網（成功率最高、速度最快）
2. 官網沒有的年份 → 請閣主從 MOPS/TWSE 手動下載
3. 所有年報存入個股資料夾的 `年報/` 子目錄
4. 命名格式：`{年度}_公司名_年報.pdf`（如 `2024_華邦電_年報.pdf`）

#### A-2. 公開說明書取得

**來源：TWSE 電子資料查詢系統**
- 查詢清單：`https://doc.twse.com.tw/server-java/t57sb01?step=1&co_id={股票代號}&mtype=B`
- 包含歷年增資/發行可轉債的公開說明書
- 下載方式（兩步驟，已實測驗證）：
  1. POST 取得動態連結：`curl -d "step=9&kind=B&co_id={代號}&filename={檔名}" https://doc.twse.com.tw/server-java/t57sb01`
     → 回傳含動態時間戳的臨時檔名
  2. GET 下載 PDF：`curl -O https://doc.twse.com.tw/pdf/{動態檔名}`
- 存入個股資料夾的 `公開說明書/` 子目錄
- 保留原始檔名（如 `202405_2344_B011.pdf`）

#### A-3. PDF → 文字檔轉換（AI 可讀化）

年報與公開說明書下載完成後，**必須轉換為純文字檔**，AI 才能直接讀取分析。
使用混合模式確保所有頁面（含掃描圖片頁）都能被轉換。

**工具組合**：
- **PyMuPDF（fitz）**：主要文字擷取引擎，速度快，處理有文字層的 PDF
- **EasyOCR + pypdfium2**：處理掃描版/圖片頁面，支援繁體中文 + 英文 OCR，GPU 加速
- 環境：Anaconda Python（`C:/ProgramData/anaconda3/python.exe`）

**執行指令**：
```
C:/ProgramData/anaconda3/python.exe pdf_to_text.py          # 混合模式（預設推薦）
C:/ProgramData/anaconda3/python.exe pdf_to_text.py --text-only  # 僅文字擷取（快速）
C:/ProgramData/anaconda3/python.exe pdf_to_text.py --ocr-only   # 全部 OCR（最完整）
```

**轉換腳本**：`pdf_to_text.py`，存放於個股資料夾根目錄，可重複執行

**混合模式流程**（每頁）：
1. PyMuPDF 嘗試擷取文字層
2. 若文字 < 30 字 → 判定為掃描/圖片頁
3. pypdfium2 渲染為圖片（150 DPI）→ EasyOCR 辨識繁體中文
4. 輸出標記來源：`[文字層]` 或 `[OCR]`

**輸出規則**：
- 每個 PDF → 對應 .txt 檔（保留原檔名，副檔名改為 .txt）
- 每頁標記頁碼和來源（`--- 第 N 頁 / 共 M 頁 [文字層/OCR] ---`）
- 輸出編碼：UTF-8
- 年報文字檔存入：`年報_文字檔/`
- 公開說明書文字檔存入：`公開說明書_文字檔/`

**OCR 注意事項**：
- OCR 速度約 15 秒/頁（GPU），大量掃描 PDF 需耗時較長
- 需設定環境變數 `KMP_DUPLICATE_LIB_OK=TRUE`（避免 OpenMP 衝突）
- OCR 結果可能有少量誤字（如「股份」→「股曹」），不影響 AI 理解大意
- 純裝飾頁（無文字無圖表）會標記為 `[無可辨識內容]`

**實測效能**（華邦電 2344）：
- 文字擷取（有文字層的頁面）：約 2 秒/頁
- OCR 辨識（掃描/圖片頁面）：約 15 秒/頁（150 DPI, GPU）
- 年報 11 個 PDF（2337 頁）：大部分為文字層，約 25 分鐘
- 公開說明書 14 個 PDF（4637 頁）：含大量掃描頁，需較長時間

#### A-4. 擷取重點

**年報擷取**（從文字檔中讀取）：
致股東報告書、營運概況、產業概況、財務分析、營收地區別、
主要客戶/供應商、研發概況、員工資訊、風險因素、關係企業、
公司治理、財報附註、會計師報告。

**公開說明書擷取**（從文字檔中讀取）：
增資用途、發行條件、資金運用計畫、風險因素、轉換條件（CB）。

### 管道 B — AI 必搜清單（使用 WebSearch）
設定檔：`天機閣-設定/投資分析/必搜清單.json`

**執行方式**：使用 Claude 的 **WebSearch 工具**逐項搜尋，取得最新即時資料。

逐項搜尋標準化清單，確保完整覆蓋：
- **必須（一次來源）**：月營收（近 12 月）、最近法說會、最近季報、公司官網產品頁、重大訊息（近 6 月）、董監持股質押
- **建議（二次來源）**：近 3 月新聞（≥5 篇）、產業研究報告、分析師報告、內部人異動、三大法人持股趨勢
- **加分（結構來源）**：供應鏈圖譜、專利資料、104/LinkedIn 徵才方向

**反面證據搜尋**：對每條公司自述的正面資訊，主動搜尋反面證據。

**WebSearch 搜尋詞組範例**（管道 B 專用）：
```
"{公司名} {代號} 法說會 {當年度}"
"{公司名} 月營收 {當年度}"
"{公司名} {代號} 重大訊息"
"{公司名} 產業報告 {產業關鍵字}"
"{公司名} 新聞 {近期月份}"
"{公司名} 董監持股 質押"
```

### 管道 C — 市場調查五條線
從五個外部視角交叉驗證公司真實競爭力：
1. **下游驗證**（客戶端）：客戶年報提到嗎？採購政策變動？終端銷售？應付帳款天數？
2. **上游驗證**（供應商端）：供應商列你為主要客戶嗎？原物料價格趨勢？
3. **競爭對手驗證**：對手營收成長率對比？毛利率趨勢？新產品/新產能？
4. **產業第三方驗證**：產業公會統計？國際研究機構預測？專利趨勢？
5. **人才市場驗證**：徵才方向？員工評價？核心人員異動？

### 管道 D — finlab 量化資料庫
設定檔：`天機閣-設定/投資分析/籌碼_finlab_mapping.json`

**直接執行程式碼取數**，不等待、不標記「待補」。需要什麼數據就即時寫 finlab 程式碼抓取。

取數範圍：
- 價量資料：收盤價、成交量、本益比
- 財務報表：月營收、季報、EPS、毛利率、現金流、資產負債表
- 籌碼資料：法人買賣超、外資持股、融資融券、股權分散、借券
- 產業比較：同產業個股清單、同業財報比較

**原則**：
- finlab 有的優先用 finlab（速度快、標準化、可回溯）
- finlab 沒有的才走管道 E 網路搜尋
- 分析過程中隨時可補充取數，不限於第一步

### 管道 E — Claude 即時網路搜尋（WebSearch）
**這是整個分析流程中最重要的即時資料來源**，貫穿所有步驟。

Claude 在分析過程中，**主動使用 WebSearch 工具搜尋最新資料**，不限於第一步。

#### 搜尋時機
| 步驟 | 搜尋目的 | 搜尋詞組範例 |
|------|---------|-------------|
| 第一步 | 基礎資料收集 | 見管道 B 搜尋詞組 |
| 第三步 | 產業面補充 | "{產業} 市場規模 {年度}"、"{公司} 競爭對手 市佔率" |
| 第三步 | 籌碼面補充 | "{代號} 外資買賣超"、"{代號} 融資融券" |
| 第四步 | ACH 假說驗證 | "{公司} 利空"、"{產業} 風險"、"{公司} 客戶轉單" |
| 第五步 | 壓力測試佐證 | "{產業} 產能過剩"、"{公司} 訴訟"、"{公司} 虧損" |
| 第六步 | 評審團補充 | "{產業} 合約報價 {季度}"、"{公司} 稼動率" |

#### 搜尋策略
1. **多角度搜尋**：同一議題至少用 2 組不同關鍵字搜尋，避免遺漏
2. **正反並搜**：搜尋正面資訊時，同步搜尋反面/負面關鍵字
3. **時效優先**：搜尋詞加入年度/季度，優先取得最新資料
4. **交叉驗證**：不同來源的數據互相比對，不一致時標記矛盾
5. **來源標註**：每條搜尋結果都標註來源網址、日期、可靠度

#### WebSearch 使用原則
- **主動搜尋，不等指令**：分析過程中發現資料缺口，立即用 WebSearch 補齊
- **搜尋結果必須標註 FIAS**：網路搜尋結果默認為 I（推論）或 A（假設），除非為官方來源可標 F（事實）
- **優先中文搜尋**：台股分析優先搜尋繁體中文內容
- **英文補充**：產業趨勢、國際市場資料用英文關鍵字補充搜尋
- **不可取代一次來源**：WebSearch 是補充工具，官方財報/年報/法說會等一次來源仍優先
- **WebFetch 深度讀取**：WebSearch 找到重要頁面後，可用 WebFetch 工具深入讀取內容

## 第二步：資料分級與整合

設定檔：`天機閣-設定/投資分析/FIAS分級規則.json`

### FIAS 四級分類
| 等級 | 定義 | 信心權重 | 自動分級規則 |
|------|------|---------|-------------|
| **事實 F** | 可獨立驗證的客觀數據 | 1.0 | 官方文件 + 交叉驗證 ≥2 來源 |
| **推論 I** | 從事實合理推導的邏輯結論 | 0.75 | 有推理鏈 + 事實基礎 |
| **假設 A** | 被當作真實但未驗證的前提 | 0.45 | 涉及未來 + 無法直接驗證 |
| **推測 S** | 缺乏充分事實支撐的猜測 | 0.15 | 基於直覺或傳聞 |

### 整合流程
1. FIAS 分級：每條資料標記等級
2. 來源可靠度：A（官方文件）→ E（未驗證傳聞）
3. 去重合併：同一事實多來源 → 合併，標記 n=多來源
4. 矛盾偵測：不同來源數字不一致 → 標記矛盾（一次來源 > 二次來源 > 推測；較新 > 較舊）
5. 時效標記：超過 6 個月的資料降級
6. 缺口報告：必搜清單覆蓋率統計
7. 會計品質掃描：M-Score（> -2.22 = 警示）、Z-Score（< 1.81 = 危險）、台股紅旗指標

### 台股紅旗指標
- 董監質押率 > 50%
- 會計師非「無保留意見」
- 近 2 年更換會計師
- 關係人交易佔營收 > 20%
- 背書保證 > 淨值 50%
- 應收/存貨週轉天數 > 同業 1.5 倍
- 中國營收佔比 > 40%

## 第三步：競爭力分析

### 3a. 競爭力地圖（7 維度）
設定檔：`天機閣-設定/投資分析/競爭力地圖模板.json`

每個維度都是「事實題」不是「感覺題」：
1. **產品事實**：營收結構、產品差異化、技術門檻、ASP 趨勢
2. **競爭事實**：競爭者清單+市佔率、成本結構比較、進入障礙
3. **需求事實**：客戶集中度、BOM 佔比、轉換成本、終端需求
4. **財務事實**：毛利率趨勢、營業利益率、自由現金流、營運效率
5. **供應鏈事實**：圖譜中心度、上下游議價力、可替代性
6. **管理層事實**：承諾兌現率、薪酬結構、董監質押、內部人異動
7. **市場調查事實**：外部五條線結果、公司自述 vs 外部矛盾清單

### 3b. 護城河框架
設定檔：`天機閣-設定/投資分析/護城河框架模板.json`

四個面向的量化評級：
- **角色定位**（0-5 分）：需求結構轉移、單位價值提升、瓶頸地位、客戶轉換成本、BOM 地位
- **定價能力**（0-14 分）：原物料壟斷、產能瓶頸、需求耐久性、替代供應商數、替代技術、合約結構、客戶付費意願
- **市佔率動態**：絕對值 + 趨勢 + 取得方式 + 可守性
- **護城河總評**：寬(≥2來源+定價權≥7) / 窄(1來源+定價權4-6) / 不確定 / 無

Morningstar 五大來源交叉驗證：無形資產、轉換成本、網路效應、成本優勢、有效規模。

### 3c. 產業面分析
設定檔：`天機閣-設定/投資分析/產業分析模板.json`

- 產業生命週期定位（萌芽/成長/成熟/衰退）
- 產業規模與成長動能（TAM、CAGR、驅動/衰退因素）
- Porter 五力量化（每力有具體數據指標和評分標準）
- 公司在產業中的定位（規模排名/市佔率/技術定位/成本定位/客戶定位）
- 同業比較表（自動生成）
- 產業趨勢與政策面

### 3d. 籌碼分析
設定檔：`天機閣-設定/投資分析/籌碼分析模板.json`

六大維度（finlab 可自動取得項目優先用 finlab）：
1. **三大法人動態**：外資/投信/自營商買賣超、持股比率（5/20/60 日）
2. **融資融券**：融資餘額+使用率、融券+券資比、借券賣出
3. **集中度與大戶結構**：股權分散表、千張大戶比率、持股人數變化
4. **內部人交易**：申報轉讓、自行買入、庫藏股、關聯交易
5. **技術面籌碼信號**：成交量趨勢、週轉率、當沖比率
6. **警示狀態**：處置股/注意股/警示股

## 第四步：競爭假說分析（ACH）

設定檔：`天機閣-設定/投資分析/ACH矩陣模板.json`

### ACH 八步驟
1. 列出所有競爭假說（≥4 個，含 ≥1 看空）
2. 列出所有證據，標記 FIAS 等級和可靠度
3. 建立 ACH 矩陣（證據 × 假說，標記 CC/C/N/I/II）
4. 計算診斷力（最高分 - 最低分，越大越能區分假說）
5. 加權計分，選「最少不一致」的假說
6. 敏感度檢查：移除前 3 條關鍵證據，結論是否改變
7. 輸出主要假說 + 信心水準 + 替代假說
8. 設定監測指標和觸發條件

## 第五步：假設壓力測試

### A. 關鍵假設檢查
列出所有假設（市場/公司/估值/資料品質），每個假設回答 5 個問題：
1. 什麼條件下不成立？
2. 過去 10 年有無反例？
3. 有哪些數據可直接驗證或反駁？
4. 如果錯了影響多大？（1-5 分）
5. 是否因「一直以來都是這樣」而假設？

三桶分類：有支撐（有數據證實）/ 需附註（部分證據）/ 無支撐（純推測，紅色警示）

### B. 事前驗屍法
核心提問：「假設 12 個月後虧損 40%，回頭看原因是什麼？」
- 強制列出 ≥7 個可能原因（涵蓋公司/產業/宏觀/地緣/估值）
- 每個原因：發生機率(1-5) × 衝擊程度(1-5) = 風險分數
- 風險分數 ≥15 → 必須有預警信號和應對方案

### C. 敏感度分析
找出「哪個假設一旦錯誤，結論全翻」的致命因子。
測試各假設錯誤時對 EPS 的影響百分比。

### D. 盲點聲明
誠實列出「本報告無法取得的關鍵資料」，說明對結論信心度的影響。

## 第七步：決策輸出（質化分析報告）

### 設計理念
本步驟的目的**不是告訴閣主要不要買**，而是：
- 把所有找到的證據攤開 — 哪裡好、哪裡不好
- 以質化分析為主軸 — 這公司有沒有優勢？有沒有值得買進的理由？目前股價有什麼風險？
- 挖掘市場可能忽略的問題與優勢
- 最終決策由閣主自行判斷

## 第七步 B：推衍預測（OSINT + 推衍能力）

### 設計理念
在決策輸出的後段、投資結論之前，運用天機閣的 **情報收集能力（OSINT）** 和 **推衍能力** 推算未來可能的變化路徑。

### OSINT 情報收集
- **使用 WebSearch 主動蒐集最新情報**，不等閣主指示
- 搜尋範圍：
  - 最新股價變化
  - 最新 EPS / 營收公布
  - 產業合約價 / 現貨價最新動態
  - 競爭對手最新動態（產能、定價、策略變動）
  - 管理層最新發言
- 情報蒐集後建立「最新情報更新表」，欄位包含：項目、分析日數據、最新情報、含義
- **每條情報必須標註來源和 FIAS 等級**
- 建立「情報可信度評估表」：情報、來源、可信度、限制

### 推衍鏈建構
依時間軸拆為三段推衍：
1. **短期（1-3 個月）**：以最近的關鍵事件（如合約價談判、季報公布）為推衍起點
2. **中期（3-6 個月）**：建立多路徑推演（至少 2-3 條路徑），每條路徑標註機率評估
3. **長期（6-12 個月）**：評估產業循環位置、結構性變化、公司長期競爭力

每段推衍的格式：
- **情報 → 直接影響 → 間接影響 → 推衍結論**
- 多路徑推演時標註觸發條件和機率
- 推衍含義用口語化表達，讓閣主容易理解

### 重要原則
- **不抄分析師資料**：不引用分析師目標價或分析師預估，用自己掌握的證據推算
- **不直接引用個別評審意見**：評審團意見應整合，不顯示評審員名字
- **推測必須標明**：推測內容標 [推測]，不可偽裝為事實

## 第七步 C：股價合理性分析

### 設計理念
用多種估值方法交叉檢驗目前股價是否合理，找出「市場在想什麼」以及「哪裡可能不合理」。
**核心問題：目前股價是過高、過低、還是合理？**

### 五種估值方法（全部使用）

#### 方法一：前瞻本益比法
- 用已驗證的最新 EPS 為錨點（如月 EPS、季 EPS）
- 建立至少三個情境：強勢維持 / 維持現狀 / 衰退
- 每個情境推算年化 EPS → 搭配合理 PE 區間 → 推算合理股價
- 與現價比較，標示偏高或偏低的百分比

#### 方法二：股價淨值比法（PB）
- 計算目前 PB
- 與歷史 10 年 PB 區間比較
- 與上一輪循環高峰 PB 比較
- 評估目前 PB 是否處於歷史極端位置

#### 方法三：分部估值法
- 將公司各業務部門拆開
- 每個部門根據其特性給予不同的 PE 倍數（穩定業務高 PE、循環業務低 PE）
- 各部門估值加總 → 合理股價區間
- 找出股價波動的主要來源（哪個部門估值變動最大）

#### 方法四：隱含預期反推法
- 假設市場給予合理 PE（如 10 倍），反推目前股價隱含的 EPS 預期
- 列出達到該 EPS 需要的條件（營收、毛利率、稼動率等）
- 評估每個條件的實現機率
- **計算所有條件同時成立的聯合機率** → 這就是市場隱含的樂觀程度

#### 方法五：歷史循環比較法
- 找出上一輪類似循環的數據（股價高點、PB 高點、EPS 高峰、循環長度）
- 與本輪對比
- 評估本輪是否已超越歷史極端值
- 引用歷史循環結束後的跌幅作為風險參考

### 綜合判斷
- 五種方法交叉比對，建立「合理價位區間彙整表」
- 列出「股價可能偏低的情境」和「股價可能偏高的理由」（正反對照）
- 給出「較有安全邊際的進場價位」
- 白話結論：用最直白的語言告訴閣主目前定價邏輯

### 重要原則
- **全部使用自己掌握的證據推算**，不引用分析師目標價
- 三情境估值（樂觀版/期望值版/悲觀版）也不引用分析師數據
- 期望值版使用 ACH 假說的機率加權計算
- 每個估值方法的結論用 tip 色塊標註（好消息用 good、壞消息用 danger、中性用 warn）

| 命中率統計 | 每年 | 統計準確度 → 回饋修正方法論 |

## 報告歸檔

每檔個股一個專屬資料夾，所有分析產出集中存放：
```
天機閣-藏經閣/投資分析資料/
├── 2344_華邦電/              ← 個股專屬資料夾
│   ├── 第一步_資料收集.html
│   ├── 第二步_FIAS分級.html
│   ├── 第三步_競爭力分析.html
│   ├── 第四步_ACH分析.html
│   ├── 第五步_壓力測試.html
│   ├── 第六步_評審團.html
│   ├── 第七步_決策輸出.html   ← 最終投資報告
│   ├── 追蹤.json              ← 監控指標和追蹤紀錄
│   ├── 年報/                  ← 年報 PDF
│   └── 歷史/                  ← 版本歷史
├── 2330_台積電/
│   └── ...
└── ...
```

**命名規則**：`{四碼代號}_{公司簡稱}`
**每步驟完成即存檔**：不是最後才存，而是每完成一步就產出對應 HTML 存入資料夾
**更新模式**：如果已有舊分析，舊版移入 `歷史/`，新版覆蓋

### 最終報告歸檔到天機錄
第七步決策輸出完成後，最終投資報告**同時歸檔到天機錄**：
```
天機閣-藏經閣/天機錄/
└── {日期}_投資分析_{公司名}{代號}.html    ← 最終報告（與其他推衍報告一起）
```
範例：`2026-03-20_投資分析_華邦電2344.html`

個股資料夾存放的是完整分析過程和原始資料（八步驟各自的產出），天機錄存放的是最終決策報告。

## 注意事項
- 分析過程中缺乏具體數據時，**先用 WebSearch 搜尋**，搜不到才明確說明
- 資料來源優先序：finlab（量化數據）→ 年報/法說會（一次來源）→ WebSearch（即時補充）→ 推測（最後手段）
- 所有推測必須標明 [S-推測]，不可偽裝為事實
- WebSearch 結果需標註來源與 FIAS 等級，不可直接當事實使用（除非為官方來源）
- 評審團的反面觀點是強制項，不可省略
- 排除 ≠ 刪除：閣主排除的主題可能是風險來源
- 報告歸檔到藏經閣/投資分析資料/，所有模組皆可讀取
- **WebSearch 是主動行為**：不需等閣主指示，分析過程中發現資料缺口即主動搜尋
